# がん保険 査定実行プロンプト

## 概要
このプロンプトは、がん保険の給付金査定をフローチャート（TSV形式）に従って段階実行するためのものです。  
**必ず `phase0 -> phase0で確定した全ルート(route) -> phase5` の順で進めること。**  
全フロー一括読込・全入力一括読込は禁止です。
---

## 実行指示
あなたは保険給付金の査定担当です。以下のフェーズ順序を厳守して査定を実行してください。

### 0. 参照ルール（最重要）
- 参照は常に「現在フェーズで許可されたファイルのみ」。
- 未選択ルートのフローチャート・入力データを読んではならない。
- `phase0`でルート確定する前に、route系ファイルを読んではならない。
- `phase0`でONになったルートは、指定順で**すべて**実行する（スキップ不可）。

### 1. phase0（ルート確定）
最初に読み込んでよいのは、次の2つのみ。
1. `inputs/No1/phase0_input.json`
2. `flowchart/がん保険改善版/がん保険TSV_phase0_簡略版.md`

実行要件:
1. phase0フローに従って、ONになった進行ルートをすべて確定する。
2. ルート実行順は`T_ROUTE_OUT`の記載順に固定する。  
   `route_admission -> route_surgery -> route_diagnosis -> route_chemo -> route_advanced -> route_discharge -> route_adult_female`
3. `確定ルート一覧`として記録する（例: `[route_admission, route_surgery, route_discharge]`）。
4. 全ルートOFFで確定できない場合は`STOP`として中断する。

### 2. ルート判定（確定ルートを全実行）
`phase0`で確定した各ルートについて、対応する入力データとフローチャートを1組ずつ読み込み、実行順にすべて判定する。

| ルート | 入力データ | フローチャート |
|---|---|---|
| route_admission | `inputs/No1/route_admission_input.json` | `flowchart/がん保険改善版/がん保険TSV_route_admission_簡略版.md` |
| route_surgery | `inputs/No1/route_surgery_input.json` | `flowchart/がん保険改善版/がん保険TSV_route_surgery_簡略版.md` |
| route_diagnosis | `inputs/No1/route_diagnosis_input.json` | `flowchart/がん保険改善版/がん保険TSV_route_diagnosis_簡略版.md` |
| route_chemo | `inputs/No1/route_chemo_input.json` | `flowchart/がん保険改善版/がん保険TSV_route_chemo_簡略版.md` |
| route_advanced | `inputs/No1/route_advanced_input.json` | `flowchart/がん保険改善版/がん保険TSV_route_advanced_簡略版.md` |
| route_discharge | `inputs/No1/route_discharge_input.json` | `flowchart/がん保険改善版/がん保険TSV_route_discharge_簡略版.md` |
| route_adult_female | `inputs/No1/route_adult_female_input.json` | `flowchart/がん保険改善版/がん保険TSV_route_adult_female_簡略版.md` |

実行要件:
1. `phase0`で確定したルートを、実行順どおりにすべて進める。
2. 現在実行中ルート以外の入力・フローチャートは参照しない（次ルート開始時に切替）。
3. 依存順を厳守する。`route_discharge`は`route_admission`完了後、`route_adult_female`は`route_admission`+`route_surgery`完了後に実行する。
4. 必要データ不足で判定不能なら`HOLD`/`STOP`で中断し、確認事項を残す。

### 3. phase5（最終算定）
確定ルートの全判定完了後に限り、次の2つを読み込む。
1. `inputs/No1/phase5_input.json`
2. `flowchart/がん保険改善版/がん保険TSV_phase5_簡略版.md`

実行要件:
1. phase0結果とルート判定結果を引き継いで最終算定を行う。
2. 算定根拠を明記する。

---

## 処理中断ルール（厳守）
フローに`STOP`または`HOLD`が出現した場合は**その時点で査定処理を必ず中断**し、出力ファイルに以下を記録すること。
1. 中断したステップID
2. 中断理由（フラグ名と発生理由）
3. 査定担当への確認内容

## 判定の進め方
- `step_id`に従って順番に判定する（ヘッダや罫線行はスキップ）。
- YES/NO判断後、`yes_next`/`no_next`に進む。
- 複数入院/手術/給付月は古い順に処理し、`step_id`に`[入院1]`等を付記する。

### 特殊記号
| 記号 | 意味 |
|---|---|
| `REJECT` | 全給付対象外として処理終了 |
| `LOOP`/`END`/`NEXT` | ループ制御 |
| `CHK` | 条件チェック（NOならスキップ） |
| `OK`/`X` | 対象/対象外として記録 |
| `STOP`/`HOLD` | **処理中断。AIは継続不可** |

### 判断理由の記載
参照データ（項目名と値）・参照根拠・判断ロジックを簡潔に記載する。定量的判断は`source:`のみでも可。
---

- 出力先: `査定結果/yyMMdd_査定結果_[案件番号].md`
- チャット上にも実行ログを出す（中間経過は出力しない。簡潔な記載のみ可）
